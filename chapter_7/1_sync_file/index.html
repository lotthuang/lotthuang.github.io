  • 第一篇 不同主机之间的文件传输

一 说明

本篇博客用于记录和总结运维工作中在不同主机之间传输文件的一些技能

二 线上不同的Linux主机之间

(一) 通过nc命令传输

  • 在目标主机上执行:
nc -l 1234 |tar -xvf -
  • 在被迁移机上执行:
tar -cvf - 目录或者文件 |nc 目标机IP 1234
  • 能实现块级别的压缩迁移,速度是最快的,遇到大量零碎小文件要同步的时候很有用。
  • Redhat系列系统和Debian系列的系统的具体命令略有不同.

(二) 通过http服务下载

  • 在被迁移机器上执行:
python -m SimpleHTTPServer  9000
  • 在目标主机上执行:
wget http://目标IP:9000/文件名 ,进行下载

(三) 其他能想到的方式

  • scp
  • ftp下载
  • rsync

三 线上主机和本机之间

博主一直使用的Mac系统,所以这边本机是指Mac系统

(一) rsync命令

rsync -avz -e 'ssh -p 58422' 目标IP:/path/to/file ./

把目标机器上的文件,copy到本地

rsync -avz -e 'ssh -p 58422' /path/to/file 目标IP:/path/to/dir

把本机文件copy到目标机器的某个目录下

(二) rz/sz

Linux主机上需要安装lrzsz包.

本地要求使用SecureCRT. 如果是Windows,还可以使用Xshell.

Mac下常用的ssh客户端iTerm2默认并不支持. 其本质是是需要2边同时支持某些文件传输协议(Xmodem/Ymodem/Zmodem)

sz 命令用于从Linux主机下载文件.

rz 命令用于上传本地文件到Linux主机.
