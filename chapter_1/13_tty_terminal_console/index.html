概述

  • 本篇概述终端(Terminal)和TTY和控制台(Console)的区别

一 基本概念

1 终端

  • TTY = Terminal

    • Terminal 最终的目标是要和操作系统的 User Process 通信, 势必先经过 Kernel。
    • 1970 年代所谓终端，其实就是一个物理设备，也就是计算机的输入输出设备。
    • Kernel 部分包括
    • 上面的 User Process 其实就是 Shell 进程

2 控制台

  • 控制台是一种特殊的终端
    • 当计算机出现异常时，其他通过远程线连接终端都不可用了，但是计算机会把错误信息显示在控制台上
    • 当计算机在单用户模式时, 信息也只能通过控制台显示

3 PTY(伪终端 )

  • pseudo terminal，有时也被称为 pty

  • 如果我们远程 telnet 到主机或使用 xterm 时不也需要一个终端交互么？是的，这就是虚拟终端 pty(pseudo-tty)

4 PTM/PTS

  • pts(pseudo-terminal slave)是 pty 的实现方法，与 ptmx(pseudo-terminal master)配合使用实现 pty。

  • 伪终端分为两部分，如上图所示，包括 master 和 slave 两部分。其中 master 实现了对不同接入方式的适配，它实现对来自不同设备或者软件消息的解析，将结果传输给 slave；而 slave 端其实就是一个虚拟终端

二 历史

  • 早期都是物理的终端设备, 比如VT102 物理终端
  • 后来都改成了软件模拟终端，比如 gnome-terminal, iTerm2 等
    • 确切的说后来 gnome 用来模拟 Xterm 的，而 Xterm 本身也是一个终端模拟器，它模拟了 VT102 物理终端
    • Linux 操作系统起来的时候默认是有 7 个虚拟终端的。这些虚拟终端并非前面所说的在桌面中打开的虚拟终端，而是通过显示器直接显示的。我们可以通过快捷键在这些终端之间进行切换，比如 Ctrl+Alt+ F1 可以切换到第一个终端，而 Ctrl+Alt+ F2 可以切换到第二个终端，以此类推 Ctrl+Alt+F7 可以切换到第七个终端。通常情况下对于图形界面来说，第七个是图形终端，其它是文字终端。

    • 图中 UART 其实是一种通信协议，类似 USB

三 互相之间的关系

  • 一个 SessionID 和一个 TTY 是绑定的,session id 和 tty 是一对一关系
  • 同一个 session 内，可以有多个后台进程组，但只能有一个前台进程组，session id 和 tty 一一对应，但是 tty 的输入，只有该 session 的前台组可以操作。也就说说，多个后台进程组，都可以调用该 tty 的输出，但是只有一个进程组可以控制它的输入。
